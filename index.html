7‑Day Portfolio Challenge

Pick one project. Keep it small. Test it. Publish it. This is the fastest path to employability.

1) College Policy Helper

Answer questions only from a policy doc. Refuse outside scope. Track failure modes.

  • Deliverable: README + eval set + refusal rules
  • Skill: trust & correctness over fluency

2) Placement FAQ Assistant

Create a placement FAQ assistant with clear boundaries and tests (hallucination + unsafe advice).

  • Deliverable: FAQ + 25–50 test questions
  • Skill: evaluation thinking

3) Hallucination Buster

Collect 25–50 answers from an LLM, label “supported vs unsupported”, and summarize failure patterns.

  • Deliverable: CSV/spreadsheet + findings
  • Skill: error analysis
Day‑by‑day plan (simple, realistic)
  1. Day 1: Pick project + define “intended use” + “must refuse” rules.
  2. Day 2: Draft an AI Launch Card (scope + risks + success metrics).
  3. Day 3: Create an eval set (25–50 questions) that tries to break your assistant.
  4. Day 4: Run tests; capture failures; categorize them (hallucination, refusal, ambiguity, etc.).
  5. Day 5: Add guardrails (policy, refusal patterns, citations, retrieval limits, etc.).
  6. Day 6: Re-run evals; measure improvement; document what changed and why.
  7. Day 7: Publish (GitHub/Notion) using the README template.
